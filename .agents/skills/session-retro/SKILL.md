# Session Retrospective

セッションの終わりに呼び出す振り返りスキル。コンテキストウィンドウ内の会話を分析し、4ステップのパイプラインで知見を抽出・記録する。

## 実行手順

以下の4ステップを順番に実行し、最後に結果を1つのファイルにまとめて保存する。

---

### Step 1: Analyze Session

コンテキストウィンドウ内の会話全体を振り返り、以下を構造化する。

- **やったこと**: タスクの一覧と結果（成功/失敗/未完了）
- **アプローチ**: どんな手順・ツール・戦略を取ったか
- **うまくいったこと**: スムーズに進んだ判断やパターン
- **うまくいかなかったこと**: 手戻り、失敗、非効率だった箇所とその原因
- **所要ステップ数の感覚**: 想定より多く/少なくかかったか

事実ベースで記述すること。推測や主観的な評価は避ける。

---

### Step 2: Extract Knowledge

Step 1 の分析結果をもとに、将来のセッションで再利用できる知見を抽出する。

抽出対象:
- **技術的知見**: ライブラリの挙動、APIの癖、ハマりポイントなど
- **プロジェクト固有の知識**: アーキテクチャ上の制約、暗黙の規約、依存関係
- **作業パターン**: うまくいった手順やワークフロー

各知見は以下の3点で記述する:
- **状況**: どういう場面で遭遇したか
- **学び**: 何がわかったか
- **次回**: 次にどうするか

知見がないセッション（単純作業など）では、無理に捻り出さず「特になし」として良い。

---

### Step 3: Improve Prompt

セッションの経験から、ルールやスキルの改善点を特定する。

**対象の探索**:
1. セッション中の手戻りや非効率から「ルール化すれば防げたこと」を探す
2. 既存の CLAUDE.md やスキルファイルを Read して現状を確認する
3. 差分として改善案を生成する

**提案と確認**:
1. 改善案を1つずつ提示する。各提案には以下を含める:
   - **何を**: 対象ファイルと変更箇所
   - **なぜ**: セッション中のどの問題を防げるか
   - **どう変える**: 具体的な変更内容（before/after）
2. AskUserQuestion で承認を求める
3. 承認された場合 → Edit で即適用
4. 却下された場合 → 却下理由を記録してスキップ

**制約**:
- 提案は最大3件。重要度の高いものから順に提示する
- 改善対象はユーザーの管理下のファイルのみ（CLAUDE.md、`.agents/skills/` 配下の自作スキルなど）。プラグインキャッシュ内のファイルは変更しない
- 既存ルールとの矛盾がないことを確認してから提案する
- 改善案がないセッションでは、無理に提案せず「特になし」として良い

---

### Step 4: Summarize Session

セッション全体の要約を生成する。後日「あのセッションで何やったっけ？」に答えられる粒度。

- 1-3行のサマリー
- 主要な成果物（作成/変更したファイル、PRなど）
- 次回やるべきことがあれば記載する

---

## 出力

### 保存先の解決

以下の順序で最初に見つかったディレクトリに保存する:
1. プロジェクトルートの `.matsuyoshi/` or `.matsuyoshi30/`
2. `$HOME/.matsuyoshi/` or `$HOME/.matsuyoshi30/`

Glob で `.matsuyoshi*/` と `$HOME/.matsuyoshi*/` を確認して、存在するディレクトリを使う。どちらも存在しない場合は `$HOME/.matsuyoshi30/` を Bash で作成して使う。

### ファイル名

`session-retro-YYYY-MM-DD-{topic}.md`

`{topic}` は Analyze Session の結果からセッションの主要タスクを英語ケバブケースで要約したもの（例: `add-auth-middleware`, `fix-ci-pipeline`）。

### テンプレート

以下のテンプレートに沿って Write で保存する。セクション順は読みやすさ優先（Summary が先頭）。

```
# Session Retrospective: {topic}

Date: {YYYY-MM-DD}
Branch: {current branch}

## Summary

{1-3行のサマリー}
{主要な成果物}
{次回やるべきこと}

## Analysis

### Tasks
- {タスク1}: {結果}
- {タスク2}: {結果}

### What Worked
- {うまくいったこと}

### What Didn't Work
- {うまくいかなかったこと、原因}

### Approach Notes
- {アプローチの特記事項}

## Knowledge

### {知見タイトル1}
- **状況**: {どういう場面で}
- **学び**: {何がわかったか}
- **次回**: {次どうするか}

## Prompt Improvements

### Applied
- {適用された改善: 何をどう変えたか}

### Skipped
- {却下された提案: 何をなぜスキップしたか}
```

保存完了後、ファイルパスをユーザーに報告する。
